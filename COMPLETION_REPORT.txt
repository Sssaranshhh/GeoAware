PROJECT OVERVIEW
================================================================================
The Flood Route Safety Navigator is a production-ready FastAPI backend that 
computes flood-aware safe routes using:
  • Machine Learning-based flood hazard prediction (RandomForest Classifier)
  • Geospatial hazard grid computation (elevation, slope, river proximity, discharge)
  • A* pathfinding algorithm with hazard-aware cost function
  • LLM-generated travel safety advisories (OpenAI/Groq/Fallback)
  • OpenStreetMap road network integration via NetworkX

COMPONENT DESCRIPTIONS
================================================================================

1. FASTAPI APPLICATION (main.py)
   ─────────────────────────────
   • Creates FastAPI application instance
   • Implements lifespan context manager for startup/shutdown
   • Loads ML model, road graph, and hazard processor at startup
   • Registers API routes
   • Provides health check endpoint
   
   Key Features:
     - Async/await support for non-blocking I/O
     - Graceful error handling with detailed logging
     - Modular router inclusion for API endpoints

2. CONFIGURATION (config.py)
   ──────────────────────────
   Manages all application settings:
   • File paths (data, models, cache directories)
   • Server configuration (host, port, debug mode)
   • Hazard thresholds and classification levels
   • A* pathfinding weights and parameters
   • LLM model configuration and API keys
   • Grid resolution and geographic bounds
   
   Default Values:
     - Host: 127.0.0.1
     - Port: 8000
     - Debug Mode: True
     - Hazard Grid Resolution: 0.01 degrees (~1km)
     - A* Hazard Weight Alpha: 2.0

3. APPLICATION STATE (state.py)
   ────────────────────────────
   Global state container for runtime objects:
   • flood_model: Trained RandomForest classifier
   • scaler: Feature normalization transformer
   • road_graph: NetworkX graph of road network
   • hazard_processor: Hazard data processor instance
   
   Purpose: Avoid reloading expensive objects on each request

4. API ROUTES (api/routes.py)
   ──────────────────────────
   Main endpoint implementation: POST /route/safe
   
   Workflow:
     1. Validate request (origin, destination, time_horizon)
     2. Generate hazard layer from discharge forecasts
     3. Find nearest road nodes to origin/destination
     4. Run A* pathfinding with hazard-aware costs
     5. Predict flood risk along the route
     6. Compute hazard exposure metrics
     7. Generate LLM-based safety advisory
     8. Build alternative routes (if requested)
     9. Return comprehensive JSON response
   
   Response Includes:
     - Primary route (coordinates, distance, ETA)
     - Hazard metrics (exposure, max, mean)
     - Safety advisory text
     - Route efficiency classification
     - Alternative routes (if requested)
     - Metadata (timestamp, snapping distance, discharge data)

5. API SCHEMAS (api/schemas.py)
   ────────────────────────────
   Pydantic models for request/response validation:
   
   RouteSafeRequest:
     - origin_lat: float (latitude)
     - origin_lon: float (longitude)
     - dest_lat: float (destination latitude)
     - dest_lon: float (destination longitude)
     - time_horizon: str ("now", "24h", "48h")
     - include_alternatives: bool (optional routes)
   
   RouteSafeResponse:
     - status: str ("SUCCESS", "ERROR")
     - primary_route: RouteInfo
     - alternative_routes: List[RouteInfo]
     - metadata: dict (computation info)

6. ADVISORY GENERATOR (api/advisory.py)
   ───────────────────────────────────
   Generates human-readable safety advisories using:
   • OpenAI/Groq LLM APIs (primary)
   • Rule-based fallback system
   
   Advisory Content:
     - Hazard assessment
     - Route safety recommendations
     - Suggested precautions
     - Alternative route suggestions
   
   Inputs:
     - Hazard exposure score (0-1)
     - Route distance and ETA
     - Discharge ratio vs baseline
     - Mean hazard value

7. ML LAYER (ml/*)
   ────────────────
   
   train_model.py:
     • Loads training dataset (or generates synthetic data if unavailable)
     • Trains RandomForest classifier (100 trees, 8 max depth)
     • Evaluates model performance (accuracy, ROC-AUC, confusion matrix)
     • Saves model and scaler artifacts
     • Computes feature importance rankings
   
   features.py:
     • Feature normalization and scaling
     • Missing value handling
     • Feature engineering (lag features, ratios)
     • Synthetic data generation for testing
   
   predict.py:
     • Loads saved model and scaler from disk
     • Runs inference on input features
     • Returns flood risk probability (0-1)
     • Handles batch and single predictions

8. ROUTING ENGINE (routing/*)
   ────────────────────────────
   
   graph_builder.py:
     • Loads OpenStreetMap roads from GeoJSON
     • Constructs NetworkX directed graph
     • Attaches hazard grid cell IDs to nodes
     • Provides nearest node lookup functionality
     • Caches processed graph for performance
   
   cost.py:
     • Computes edge traversal cost
     • Base cost: edge distance in km
     • Hazard penalty: (1 + alpha * hazard_score)
     • Formula: cost = distance_km * (1 + alpha * hazard)
   
   pathfinder.py:
     • Implements A* search algorithm
     • Heuristic: Haversine distance to destination
     • Uses hazard-aware cost function
     • Computes multiple paths if requested
     • Returns:
       - Path node sequence
       - Total distance in km
       - ETA in minutes (assuming 60 km/h)
       - Hazard exposure along route

9. HAZARD PROCESSING (hazard/*)
   ──────────────────────────────
   
   forecast_processor.py:
     • Parses discharge forecast data
     • Normalizes against historical baseline (850 m³/s)
     • Generates discharge ratio (0-1 scale)
     • Computes hazard input for next 24-48 hours
     • Handles missing data gracefully
   
   hazard_layer.py:
     • Combines multiple hazard factors:
       1. Elevation (lower = higher risk)
       2. Slope (flatter = higher risk)
       3. River proximity (closer = higher risk)
       4. Discharge ratio (higher = higher risk)
     • Computes hazard score per grid cell (0-1)
     • Outputs hazard grid for pathfinding
     • Uses weighted combination of factors

10. DATA SCRIPTS (scripts/*)
    ────────────────────────
    
    preprocess_grids.py:
      • Creates synthetic elevation grid (1000m-2000m range)
      • Computes slope from elevation data
      • Calculates river proximity (distance to nearest river)
      • Saves all grids as numpy arrays for fast loading
    
    download_osm.py:
      • Downloads OpenStreetMap road data via Overpass API
      • Filters for motorable roads (primary, secondary, tertiary)
      • Converts to GeoJSON format
      • Saves to data/osm/roads.geojson
    
    download_forecast.py:
      • Fetches flood forecast data from APIs
      • Processes discharge measurements
      • Caches data locally for performance

================================================================================
EXECUTION FLOW
================================================================================

1. SERVER STARTUP
   ──────────────
   • main.py creates FastAPI app
   • lifespan context manager executes:
     - Load ML model and scaler from models/
     - Load road graph from data/osm/
     - Initialize hazard processor
   • Attach API routes
   • Start Uvicorn ASGI server

2. HEALTH CHECK REQUEST
   ─────────────────────
   GET /health
   → FastAPI executes health_check()
   → Return 200 OK with status

3. SAFE ROUTE REQUEST
   ───────────────────
   POST /route/safe with JSON body
   
   Step A: Validation
     • Pydantic validates request schema
     • Check time_horizon is valid ("now", "24h", "48h")
     • Check coordinates are within bounds
   
   Step B: Hazard Processing
     • HazardProcessor prepares discharge forecasts
     • Normalizes discharge ratio (0-1)
   
   Step C: Hazard Layer Generation
     • HazardLayer combines factors:
       - Load elevation grid from data/grids/elevation.npy
       - Load slope grid from data/grids/slope.npy
       - Load river proximity from data/grids/river_proximity.npy
       - Fetch discharge_ratio from forecast processor
     • Compute hazard_score per grid cell
     • Output hazard_grid for pathfinding
   
   Step D: Road Network Snapping
     • Snap origin to nearest road node (Haversine distance)
     • Snap destination to nearest road node
     • Store snapping distances in metadata
   
   Step E: Pathfinding
     • FloodAwarePathfinder.find_path(start_node, end_node, hazard_grid)
     • A* algorithm:
       - Open set: priority queue of nodes to explore
       - Closed set: visited nodes
       - f(n) = g(n) + h(n)
         • g(n) = actual cost from start
         • h(n) = heuristic to goal (Haversine)
       - Repeat until destination reached
     • Return path (list of nodes)
   
   Step F: Path Feature Computation
     • Extract path coordinates
     • Calculate total distance (sum of edge lengths)
     • Calculate ETA (distance / 60 km/h)
     • Look up hazard values for each cell
     • Compute:
       - mean_hazard = average hazard along route
       - max_hazard = maximum hazard encountered
       - hazard_exposure = integral of hazard along path
   
   Step G: ML Flood Risk Prediction
     • Extract features from grid cells along route:
       - elevation, slope, river_proximity, discharge_ratio, etc.
     • Normalize features with scaler
     • Run RandomForest.predict_proba()
     • Average flood risk probability across route
     • Store flood_risk_score (0-1)
   
   Step H: LLM Advisory Generation
     • AdvisoryGenerator receives:
       - hazard_exposure, max_hazard, mean_hazard
       - distance_km, eta_minutes
       - discharge_ratio
       - flood_risk_score
     • Try OpenAI/Groq API call
     • On failure, use rule-based fallback
     • Generate 2-3 sentence safety recommendation
   
   Step I: Efficiency Classification
     • hazard_exposure < 0.2 → "EXCELLENT"
     • hazard_exposure < 0.35 → "GOOD"
     • hazard_exposure < 0.55 → "MODERATE"
     • hazard_exposure >= 0.55 → "RISKY"
   
   Step J: Alternative Routes (if requested)
     • Run A* again with penalty on first path edges
     • Find second-best path
     • Repeat for third path if desired
     • Return all alternatives in response
   
   Step K: Response Assembly
     • Build RouteInfo objects for each path
     • Compile metadata
     • Return RouteSafeResponse (200 OK)

TROUBLESHOOTING
================================================================================

Issue: ModuleNotFoundError: No module named 'flood_navigation'
Solution: Run from project root, use python -m for scripts

Issue: No module named 'ml'
Solution: Use full package path: from flood_navigation.ml.predict import ...

Issue: OSError: [Errno 2] No such file or directory: 'data/osm/roads.geojson'
Solution: Run preprocess_grids.py and download_osm.py first

Issue: Model not found
Solution: Run ml/train_model.py to create models/

Issue: LLM advisory fails
Solution: Check GROQ_API_KEY environment variable, fallback will auto-activate

Issue: High latency (>5 seconds)
Solution: Graph too large, reduce road coverage area, add caching

Issue: Out of memory
Solution: Reduce grid resolution, partition graph into regions
